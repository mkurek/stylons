Action Class¶

class stitems.Action(action=None, url=None, target=None)¶

Bases: dict

Action component Defines actions performed i.e. after click on the button, click on the list and so on

Parameters:
  • action (String) –

    action to be executed. Proper values are:

    • "load"
    • "send"
    • "popupShow"
    • "popupHide"
    • "clear"
  • url (String) – URL to load/send data
  • target (String) – url of component, which this action affect. This parameter is useful for load action - set as form url, and for clear action - form url also.

class stitems.Button(ui=None, text=None, icon=None, badge=None, flex=None, events=None, disabled=None, alwaysReload=None, cssClass=None)¶

Sencha Touch Button component. Defines primary button appearance and events.

Parameters:
  • ui (String) –

    user interface description. Should be a combination of:

    • "normal"
    • "back"
    • "round"
    • "forward"
    • "action"
    • "confirm"
    • "decline"

    and

    • ""
    • "small"
    • "round"
  • text (String) – text displayed on the button
  • icon (String) – icon of the button
  • badge (String) – badge (little icon on top) of the button
  • flex (Integer) – TO DO
  • disabled (Boolean) – True to make button disabled
  • events (Dictionary) – dictionary (of lists) of Action instances.

Examples:

Button( ui = "normal-round", 
        text = "SomeButton", 
        events = {
            "tap":[
                Action(action = "popupHide"), 
                Action(action = "load", url = "con/act")
            ]
        }
)

Button( ui = "confirm", 
        text = "SomeOtherButton", 
        badge = "5",
        events = {
            "tap" : Action(action = "send", dataId = "form", url = "u/r/l"
        }
)

Layout Class¶

class stitems.Layout(composition=None, align=None, pack=None)¶

Bases: dict

Sencha Touch Layout. Describe composition, align and pack of content. Layout is used for screen instances. Params align and pack are available only for “vbox” and “hbox”.

Parameters:
  • composition (String) –

    define composition of elements, you can use:

    • "auto" - Sencha Touch will choose automatically
    • "fit" - for one element, fit to all screen
    • "vbox" - arrange elements vertically down container
    • "hbox" -arranges items horizontally across container
  • align (String) –

    dpecifies the alignment of elements, available aligns:

    • "center" - aligned to the center of container
    • "end" - aligned to the end of container (right or bottom)
    • "start" - aligned to the start of container (left or top)
    • "stretch" - elements are stretched through container
  • pack (String) –

    also specifies the alignment of elements, available:

    • "center" - aligned to the center of the container.
    • "end" - aligned to the right of the container.
    • "justify" - justified with left and right of the container.
    • "start" - aligned to the left of the container.

Example:

Layout(composition = "vbox", align = "start", pack = "justify")

List Class¶

class stitems.List(itemTpl=None, items=None, emptyText=None, fullscreen=None, events=None)¶

Bases: stitems.component.Component

Sencha Touch List component. Component displaying data items as list and enabling events.

Parameters:
  • itemTpl (String) – define how to format data (look at examples)
  • items (List of Dictionaries) – list of all items
  • emptyText (String) – html with message if items is empty
  • fullscreen (Boolean) – set component to fullscreen (even over its container)
  • events (Dictionary) –

    define actions for lists events. Available:

    • "itemtap" - called when item was tapped
    • "itemdoubletap" - called when item was double tapped
    • "itemswap" - called when item was swapped

    For each event you can use ‘{index}’ in Action declaration. It will be replaced by dispatcher with index, which was source of event.

Examples:

List(itemTpl = "{name} {surname}",
     items = [{ "name" : "John",
                "surname" : "Smith" },
              { "name" : "Jimmy",
                "surname" : "Wallet" },
              { "name" : "Jack",
                "surname" : "Willis" }])
               

date1 = dict()
date1['day'] = 5
date1['month'] = 08
date1['year'] = 1955

date2 = dict(day=12, month=5, year=1820)

List(itemTpl = "{day}-{month}-{year}",
     items = [ date1, date2 ])

Screen Class¶

class stitems.Screen(fullscreen=None, margin=None, padding=None, layout=None, ui=None, alwaysReload=None, cssClass=None)¶

Bases: stitems.component.Component

Component defining basic panel properties. Designed to be used as a container for all other components.

Parameters:
  • fullscreen (Boolean) – Force the component to take up 100% width and height available.
  • margin (Integer or String) – Specifies the margin for this component. Can be a single integer (applied to all sides) or string with 1-4 numbers (css-style)
  • padding (Integer or String) – Specifies the padding for this component. Can be a single integer (applied to all sides) or string with 1-4 numbers (css-style)
  • layout – Specifies the layout for this component. For more read Layout Class specification
  • ui – Predefined ui styles for component. Acceptable values are 'dark' and 'light'

Examples:

Screen(fullscreen=True, margin=4, layout=Layout(composition=fit))

Screen(fullscreen=False, padding="1 2 3")
